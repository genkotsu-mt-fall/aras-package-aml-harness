この prompt の前に挿入されている実行コンテキストの `対象タスクディレクトリ` と `レビュー番号` を使ってください。

以下を読んでください。

- 対象タスクディレクトリの change-plan.md
- 対象タスクディレクトリの test-cases.md
- 対象タスクディレクトリの test-code-review<レビュー番号>.md
- tests 配下
- samples 配下

## 目的

テストコードレビューの指摘を反映して、unittest と必要なサンプル XML を修正してください。

## 更新してよいもの

- tests 配下
- samples 配下

## 制約

- aml_harness 配下を変更しない。
- 実装コードを修正しない。
- 外部依存パッケージを追加しない。
- change-plan.md の非対象を勝手に対象へ含めない。
- レビュー指摘を無視しない。
- 特定のアイテムタイプに依存するテストを tests/test_base.py にまとめない。
- tests/test_base.py は AML 共通エンベロープ検証のテストに留め、薄く保つ。

## 完了条件

- test-code-review<レビュー番号>.md の指摘が反映されていること。
- test-cases.md の各テストケースと unittest の対応が分かること。
- テストファイルの分割が対象ルールの粒度に合っていること。
- 必要なサンプル XML が最小構成で追加または修正されていること。

## 作業後に実行すること

```powershell
python -m unittest
```

失敗した場合も、どのテストがどの理由で失敗したかを説明してください。
